# 二 、FreeRTOS

## 一、创建任务

<img src="C:/Users/ASUS/AppData/Roaming/Typora/typora-user-images/image-20260426134020522.png" alt="image-20260426134020522" style="zoom: 90%;" />

**TaskName: 任务名称**

**Priority: 任务优先级，同级别时间片轮转，高级别抢占低级别，IDLE--> Realtime**

**Stack Size： 单位Word(4B),任务堆栈大小**

**Entry Function: 任务入口，函数名**

**Code: 选择任务函数类型{Defalut : 默认在rtos.c实现，Weak：可重写覆盖，Extern:在外部文件实现即可}**

**Parameter: 任务函数的参数指针**

**Allocation： 空间分配{Dynamic : 动态，不用管，Static: 定义Buffer}**



## 二、任务实现

### 1.基本原理

**把主函数的While(1)拆分为多个，通过调度系统实现多个任务的切换实现伪多进程（注意：此时main中的while永远不会执行）**

*高级原理见*[一、 FreeRTOS 实时操作系统](./资料/一、 FreeRTOS 实时操作系统.md)  [大疆FreeRTOS原理讲解](./资料/RoboMaster开发板C型嵌入式软件教程文档(1).pdf#page=221)

### 2.任务函数基本写法

```c
void _task(void const * argument)//和main函数写法一样
{
    Init(); //初始化，只执行一次
    
    While(1)
    {
        /*业务代码，不要Return！！！！*/ 
       
        
        osdalay(100); //把任务阻塞，不让任务一直占用CPU
        /*vtaskdalay 与 osdelay 一样，前者是freertos原生，后者为cubemx封装*/
    }
   
}
```

### 3.队列发送

**实现原理：**

**发送方将数据打包，塞入队列，接收方打开队列获得数据包**

**可以减少因rtos轮转而导致丢包情况**

*使用示例：*

```c
/* 发送方 */
//****
/*发送变量打包 ***/
xQueueSend(Can1MsgHandle, &pkt, portMAX_DELAY);  /* （队列句柄，变量地址，等带时间）*/



/* 接收方 */
FloatPacket_t rcvPkt;/*接收缓存数据包 */
float out1, out2, out3, out4;/* 解包后的变量*/
       if(xQueueReceive(Can1MsgHandle, &rcvPkt, portMAX_DELAY) == pdPASS)/*等待收到新数据*/
        {
            // 解包
            unpack_float(rcvPkt, &out1, &out2, &out3, &out4);
      
        }
```
